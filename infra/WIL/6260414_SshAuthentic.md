# SSH 인증방식  
- Password  
  - 말 그대로 비밀번호를 입력해서 접속한다, 비밀번호만 알면 누구나 접속 할 수 있기 때문에 보안적으로 가장 좋지 않음  
- SSH Key  
  - ssh-keygen 명령어를 통해 pem 키를 생성하고 pub키를 서버의 /etc/sshd_config.d/authorized_keys 같은 파일에 저장하면 이후 password 입력 없이 키 인증을 통해 ssh 접속을 할 수 있다.  
- 인증서
  - SSH Key 방식과 비슷하게 pem키와 pub키를 생성하고 pub키를 서버에서 신뢰하는 ca에게 서명받아서 접속시 인증서 + pem키로 인증을 하는 방식이다.  
  - 유저 추가를 직접 안해도 되고 자동 만료가 된다는 것이 장점  
    - 솔직히 개념이 조금 어렵다..대충은 알겠는데 이해하려면 실습을 해보는게 좋을듯
  
## ed25519 vs RSA  
- ed25519는 256bit를 주로 사용하며 rsa 보다 적은 양의 연산을 사용하는 걸 보여주면서 보안성도 오히려 rsa보다 높은 모습을 보인다 
- 소인수 분해 기반인 rsa와 달리~~ 타원곡선 암호화(**ECDSA** 1학년때 했었는데)를~~ (공부하면서 더 찾아봤는데 ECDSA가 아닌 EdDSA기반이라고 한다...ECDSA보다 빠르고 몇가지 차이점도 있음 recover 등)  기반으로 작동한다  
  - 그로 인해 연산에는 RSA가 더 오랜 시간이 걸리지만 대부분 ed25519의 안정성이 더 높은 듯 해보인다
  - 왜 RSA를 쓰는지 모르겠지만... 연산이 오래걸리니 쓰는 사람도 있지 않을까 싶기도 하고, 대부분은 ed25519를 쓰는 편이 나아보이긴 한다
